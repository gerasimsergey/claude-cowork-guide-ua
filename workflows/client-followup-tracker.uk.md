# Воркфлоу: Трекер Роботи з Клієнтами

> **Орієнтовний час**: 25 хвилин
> **Складність**: Початківець
> **Категорія**: 📋 Адміністративні
> **Професії**: Усі (для малого бізнесу та ФОП)

🌐 **Мови**: [English](client-followup-tracker.en.md) | [Français](client-followup-tracker.md) | [Українська 🇺🇦](client-followup-tracker.uk.md)

## Кейс використання

Вам потрібно систематично відстежувати потенційних клієнтів (лідів) та історію спілкування з ними, щоб не втрачати можливості через забудькуватість. Цей воркфлоу допоможе вам:

- Створити просту CRM-систему на базі Excel/Google Sheets.
- Відстежувати статус кожного клієнта у "лійці продажів" (pipeline).
- Планувати та контролювати наступні кроки (follow-ups).
- Аналізувати конверсію (скільки запитів перетворюються на замовлення).

> ⚠️ **Важливо**: Це легке рішення для малого бізнесу. Якщо у вас команда >5 людей або дуже складні цикли продажів, краще використовувати професійні CRM (HubSpot, Pipedrive тощо).

## Покрокова інструкція

### Крок 1: Створення структури трекера

```bash
mkdir -p ~/Cowork-Workspace/{input/contacts,output/tracking}
```

### Крок 2: Генерація таблиці трекера

Попросіть Cowork створити файл:

```
Створи систему відстеження клієнтів (Client Follow-up Tracker):

ФАЙЛ: ~/Cowork-Workspace/output/tracking/client-tracker.xlsx

ЛИСТ 1: "Активні ліди"
Колонки:
| ID | Дата додавання | Компанія/Ім'я | Контактна особа | Телефон | Email | Джерело (Звідки прийшов) | Статус | Остання взаємодія | Наступний крок | Дедлайн | Пріоритет | Нотатки | Очікувана сума |

ЛИСТ 2: "Етапи воронки"
Етапи:
1. Новий запит
2. Перший контакт встановлено
3. Кваліфіковано (потреба підтверджена)
4. Кошторис надіслано
5. Переговори/Узгодження
6. Виграно (Клієнт підписав)
7. Втрачено (Відмова)
8. Відкладено (На майбутнє)

Формат: Excel з випадаючими списками та умовним форматуванням (просрочені дедлайни — червоним).
```

### Крок 3: Імпорт існуючих контактів

```
Імпортуй контакти з файлу: ~/Cowork-Workspace/input/contacts/leads.csv

Для кожного контакту:
- Надай ID (L001, L002...).
- Встанови статус "Новий запит".
- Встанови дедлайн для наступного кроку: [Сьогодні + 2 дні].
- Пріоритет: "Середній".
```

### Крок 4: Встановлення правил "фоллоу-апів" (Follow-up Rules)

```
Налаштуй автоматичні нагадування в трекері:

- ПРАВИЛО 1: Новий запит → Контакт протягом 24 годин.
- ПРАВИЛО 2: Кошторис надіслано → Передзвонити/написати через 3 дні, щоб дізнатися рішення.
- ПРАВИЛО 3: Відмова → Запитати причину та позначити для розсилки через 6 місяців.

Додай ці правила як підказки в колонку "Наступний крок".
```

### Крок 5: Щоденний план дій

Щоранку просіть Cowork проаналізувати трекер:

```
Проаналізуй трекер ~/Cowork-Workspace/output/tracking/client-tracker.xlsx

Згенеруй план дій на сьогодні [Дата]:
1. 🚨 ТЕРМІНОВО (Прострочені): Список контактів, кому мали зателефонувати вчора.
2. 📅 НА СЬОГОДНІ: Кому надіслати кошторис, кого привітати, кому нагадати про себе.
3. 📊 СТАТИСТИКА: Загальна сума в Pipeline, кількість нових лідів за тиждень.

Збережи як: ~/Cowork-Workspace/output/tracking/daily-plan-[date].pdf
```

### Крок 6: Фіксація результатів розмови

Після кожного дзвінка або імейла оновлюйте дані:

```
Онови статус клієнта L042 (Іван Іванов):
- Остання взаємодія: [Сьогодні], Телефонний дзвінок.
- Результат: Погодився на зустріч.
- Наступний крок: Виїзд на об'єкт.
- Дедлайн: [Дата].
- Статус: З "Перший контакт" на "Кваліфіковано".
```

## Найкращі практики

1. **Оновлюйте негайно** — Записуйте результат розмови одразу після її завершення, поки пам'ять свіжа.
2. **Будьте наполегливими, але не нав'язливими** — Стандартний цикл фоллоу-апів: 1 день, 3 дні, 7 днів, 14 днів. Якщо після 4-го контакту немає відповіді — переносьте в "Відкладено".
3. **Фіксуйте "Болі"** — У нотатках записуйте не тільки ціну, а й те, що важливо клієнту (наприклад: "дуже важливі терміни до Великодня"). Використовуйте це у наступних розмовах.
4. **Аналізуйте причини відмов** — Додайте колонку "Причина відмови" (Дорого, Довго, Обрали іншого). Це допоможе вам покращити свій бізнес.
5. **Дати — це закон** — Кожен активний клієнт ПОВИНЕН мати заповнену дату наступного кроку. Клієнт без дати — це втрачений клієнт.
