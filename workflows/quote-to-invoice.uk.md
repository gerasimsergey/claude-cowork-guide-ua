# Воркфлоу: Перетворення Кошторис → Інвойс

> **Орієнтовний час**: 10 хвилин
> **Складність**: Початківець
> **Категорія**: 📋 Адміністративні
> **Професії**: Усі

🌐 **Мови**: [English](quote-to-invoice.en.md) | [Français](quote-to-invoice.md) | [Українська 🇺🇦](quote-to-invoice.uk.md)

---

## Кейс використання

Клієнт прийняв ваш кошторис, і вам потрібно швидко перетворити його на інвойс (рахунок). Цей воркфлоу допоможе вам:

- Автоматично перетворювати кошторис на інвойс
- Підтримувати послідовність нумерації та сум
- Додавати необхідні юридичні реквізити інвойсу
- Правильно архівувати документи

> ⚠️ **Важливо**: Завжди перевіряйте, чи прийнятий кошторис точно відповідає послугам, на які виставляється рахунок.

---

## Попередні вимоги

- [ ] Cowork увімкнено в Claude Desktop
- [ ] Прийнятий кошторис (PDF або Word)
- [ ] Наступний номер інвойсу
- [ ] Підтвердження від клієнта (email, підпис або усне підтвердження)
- [ ] Створена робоча папка

## Покрокова інструкція

### Крок 1: Підготовка вихідного кошторису

```bash
mkdir -p ~/Cowork-Workspace/{input,output/invoices}

# Копіювання прийнятого кошторису
cp ~/Documents/Quotes/Q2026-042.pdf ~/Cowork-Workspace/input/
```

### Крок 2: Перетворення на інвойс

```
Перетвори кошторис ~/Cowork-Workspace/input/Q2026-042.pdf на інвойс

ТРАНСФОРМАЦІЇ:
1. Заміни "КОШТОРИС" на "ІНВОЙС" (або "РАХУНОК-ФАКТУРА")
2. Заміни номер кошторису Q2026-042 на INV-2026-085
3. Заміни дату кошторису на дату інвойсу: 20 січня 2026 року
4. Додай термін оплати: 19 лютого 2026 року (30 днів)

ЗАЛИШИ БЕЗ ЗМІН:
- Дані клієнта
- Послуги та кількість
- Ціни за одиницю та підсумки (проміжний підсумок, податок, разом)
- Загальні положення та умови (якщо є)

ДОДАЙ:
- Примітку: "Кошторис Q2026-042 від [дата кошторису] прийнятий [дата прийняття]"
- Умови оплати: Net 30 днів
- Пеню за несвоєчасну оплату: [відповідна ставка]
- Спосіб оплати: [Банківський переказ / Чек / Інше]
- Банківські реквізити (IBAN/номер рахунку)

ВИДАЛИ (якщо є):
- Згадку "Кошторис не є обов'язковим до виконання"
- Термін дії кошторису (більше не актуально для інвойсу)

Формат: PDF
Зберегти до: ~/Cowork-Workspace/output/invoices/INV-2026-085.pdf
```

### Крок 3: Перевірка узгодженості

Cowork відобразить план. **Перевірте**:

- ✓ Ідентичність сум у кошторисі та інвойсі
- ✓ Правильність та унікальність нумерації
- ✓ Узгодженість дат (дата інвойсу ≥ дати кошторису)
- ✓ Наявність усіх юридичних реквізитів

Якщо в кошторис були внесені зміни:

```
УВАГА: Клієнт попросив внести зміни після затвердження кошторису

Оригінальний кошторис Q2026-042:
- Послуга А: $500

Запит клієнта:
- Послуга А: $600 (додано преміум-опцію)

Інвойс INV-2026-085:
- Послуга А преміум: $600
- Додати примітку: "Зміна узгоджена з клієнтом [дата]"
```

### Крок 4: Генерація та архівування

```
Продовжуй перетворення
```

Перевірте та архівуйте:

```bash
# Відкрити згенерований інвойс (macOS)
open ~/Cowork-Workspace/output/invoices/INV-2026-085.pdf

# Створити зв'язок кошторис ↔ інвойс для відстеження
echo "Кошторис Q2026-042 → Інвойс INV-2026-085 від 20.01.2026" >> ~/Cowork-Workspace/output/correspondences.txt
```

### Крок 5: Подальші дії (опціонально)

Створення нагадування про термін оплати:

```
На основі інвойсу INV-2026-085 (термін оплати 19.02.2026):

Створи нагадування про оплату:
- Дата сповіщення: 12.02.2026 (за 7 днів до терміну)
- Дія: Перевірити отримання оплати
- Якщо не отримано: Надіслати ввічливе нагадування

Формат: Рядок у ~/Cowork-Workspace/output/payment-reminders.csv
Стовпці: Інвойс | Клієнт | Сума | Термін оплати | Нагадування | Статус
```

## Приклади промптів

### Просте перетворення

```
Кошторис Q2026-038 прийнято → Інвойс INV-2026-074

Кошторис: ~/Cowork-Workspace/input/Q2026-038.pdf
Дата інвойсу: 20.01.2026
Термін оплати: +30 днів (19.02.2026)
Без змін у переліку послуг

Додай:
- Примітку "Кошторис Q2026-038 від 12.01.2026 прийнятий 18.01.2026"
- Банківські реквізити: IBAN... SWIFT...
- Оплата банківським переказом

Зберегти до: ~/Cowork-Workspace/output/invoices/INV-2026-074.pdf
```

### Перетворення з депозитом (авансом)

```
Кошторис Q2026-051 Загальна сума: $5,000

Клієнт сплачує депозит 30% = $1,500

Згенеруй 2 інвойси:

ІНВОЙС 1 (депозит):
- Номер: INV-2026-089
- Дата: 20.01.2026
- Сума: $1,500
- Примітка: "Депозит 30% - Кошторис Q2026-051"
- Термін: Негайно (оплата при замовленні)

ІНВОЙС 2 (залишок - виписати пізніше):
- Номер: INV-2026-XXX (буде визначено)
- Сума: $3,500
- Примітка: "Залишок - Кошторис Q2026-051 - Депозит INV-2026-089 вирахувано"
- Термін: Після доставки/завершення робіт

Наразі згенеруй лише INV-2026-089 (депозит)
Зберегти до: ~/Cowork-Workspace/output/invoices/INV-2026-089-deposit.pdf
```

### Пакетне перетворення

```
Перетвори групу з 5 прийнятих кошторисів у ~/Cowork-Workspace/input/accepted-quotes/

Для кожного кошторису:
1. Прочитай номер кошторису Q2026-XXX
2. Згенеруй інвойс INV-2026-YYY (послідовна нумерація, починаючи з INV-2026-090)
3. Дата інвойсу: 20.01.2026
4. Термін оплати: +30 днів
5. Додай стандартні реквізити

Також створи зведений файл Excel:
- Стовпці: Кошторис | Інвойс | Клієнт | Сума | Дата | Термін оплати
- Зберегти до: ~/Cowork-Workspace/output/conversions-january-2026.xlsx
```

## Усунення несправностей

### Неповний або змінений кошторис

**Причина**: Переговори після надання кошторису
**Рішення**: Задокументуйте зміни:

```
Кошторис Q2026-042 був змінений після прийняття:

ОРИГІНАЛЬНИЙ КОШТОРИС:
- Послуга А: $1,000
- Послуга Б: $500
Разом: $1,500

УГОДА З КЛІЄНТОМ (email від 18.01):
- Послугу А видалено
- Послуга Б: $500
- Додано Послугу В: $700
Разом: $1,200

ІНВОЙС INV-2026-085:
Використовуй фінальну угоду ($1,200)
Додай примітку: "Ціна згідно з домовленістю по email від 18.01.2026"
Додай копію листа про прийняття
```

### Пропуск у нумерації

**Причина**: Забутий або дубльований інвойс
**Рішення**: Перевірте реєстр:

```
Перелічи всі інвойси в ~/Cowork-Workspace/output/invoices/
Відсортуй за хронологічним номером
Визнач останній використаний номер: INV-2026-084
Наступний номер: INV-2026-085

Якщо в нумерації є пропуск (напр: INV-2026-082 відсутній):
→ Розслідуй: інвойс скасовано? забуто?
→ Ніколи не використовуй скасований номер повторно (якщо потрібно, створи INV-2026-085A)
```

## Найкращі практики

1. **Зберігайте кошториси** — Архівуйте кошторис ТА інвойс разом (для простежуваності)
2. **Формальне прийняття** — Завжди майте підтвердження прийняття (email, підпис)
3. **Розумний термін** — Виставляйте інвойс не пізніше ніж через 1 місяць після прийняття кошторису
4. **Узгодженість сум** — Якщо сума змінилася, задокументуйте це та повідомте клієнта
5. **Унікальна нумерація** — Ніколи не створюйте 2 інвойси з однаковим номером
6. **Подвійна перевірка** — Перевірте перед надсиланням (помилка в інвойсі = кредит-нота + новий інвойс)
7. **Чіткий термін оплати** — Завжди вказуйте крайній термін платежу
8. **Оперативність** — Надсилайте інвойс протягом 48 годин після прийняття кошторису (професіоналізм)
