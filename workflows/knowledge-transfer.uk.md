# Воркфлоу: Передача знань

> **Орієнтовний час**: 2-4 години (залежно від складності професії)
> **Складність**: Просунута
> **Категорія**: 🗂️ Організація
> **Професії**: Підрядники, Власники малого бізнесу

🌐 **Мови**: [English](knowledge-transfer.en.md) | [Français](knowledge-transfer.md) | [Українська 🇺🇦](knowledge-transfer.uk.md)

---

## Кейс використання

Вам потрібно передати свій професійний досвід співробітнику, учню або наступнику. Цей воркфлоу допоможе вам:

- Формалізувати приховані знання та професійні хитрощі
- Створити структуровані навчальні документи
- Зберегти унікальне ноу-хау бізнесу
- Прискорити розвиток навичок нових працівників

> ⚠️ **Критичний ризик**: 60% малих ремісничих підприємств втрачають ключові ноу-хау зі звільненням досвідчених працівників. Документація = стійкість бізнесу.

## Покрокова інструкція

### Крок 1: Картування ноу-хау

```
Допоможи мені скласти карту мого професійного ноу-хау для передачі:

МОЯ ПРОФЕСІЯ:
[Напр: Сантехнік, Пекар, Електрик, Тесляр...]

СФЕРА ПЕРЕДАЧІ:
[Напр: Встановлення конденсаційних котлів, Рецепт листкового тіста, Трифазні щити...]

РІВЕНЬ ОТРИМУВАЧА:
[Початківець / Середній / Просунутий]

КОНТЕКСТ ПЕРЕДАЧІ:
[Навчання учня / Передача справ перед виходом на пенсію / Адаптація нового працівника]

ЗАПИТУВАНА МАПА:
1. Теоретичні знання (що треба знати).
2. Практичні навички (що треба вміти робити).
3. Професійні хитрощі (секрети майстерності).
4. "М'які" навички професії (спілкування з клієнтами, безпека).
5. Зовнішні ресурси (постачальники, нормативи).

Формат: Ментальна карта, структурована за рівнем складності.
```

### Крок 2: Документування ключових процедур

Для кожного важливого аспекту ноу-хау:

```
Створи детальну інструкцію для: [Назва процедури]

СТРУКТУРА ІНСТРУКЦІЇ:
- Мета (очікуваний результат).
- Безпека та запобіжні заходи (ризики та ЗІЗ).
- Необхідне обладнання (інструменти та розхідні матеріали).
- Попередні перевірки (що зробити перед початком).
- Покрокова процедура (з часом виконання кожного кроку).
- Поради експерта (хитрощі та як уникнути помилок).
- Фінальний контроль якості.
- Що робити в разі виникнення проблем (таблиця несправностей).

Зберегти у: ~/Cowork-Workspace/transfer/procedures/[назва-процедури].pdf
```

### Крок 3: Створення плану прогресивного навчання

```
Розроби план поступового навчання для: [Профіль отримувача]

СТРУКТУРА ШЛЯХУ:
- Етап 1: Основи (Тижні 1-4) — Теорія та спостереження.
- Етап 2: Практика під наглядом (Тижні 5-12) — Виконання базових дій.
- Етап 3: Поступова автономність (Тижні 13-26) — Самостійна робота над стандартними кейсами.

Для кожного етапу визнач:
- Цілі та результати.
- Модулі для вивчення.
- Критерії оцінювання.

Зберегти у: ~/Cowork-Workspace/transfer/training-path.pdf
```

### Крок 4: Відео та візуальні матеріали

Для складних технічних маніпуляцій:
- **Короткі відео** (2-5 хв) зі структурою: вступ, покрокове виконання, фінальний результат, помилки.
- **Фото-інструкції** (8-12 послідовних фото з анотаціями).
- **Технічні схеми** (для абстрактних концепцій або ланцюгів).

### Крок 5: Створення бази знань

Організуйте всі матеріали в чітку структуру папок:
- `01-fundamentals/` (термінологія, норми, безпека).
- `02-procedures/` (інструкції за категоріями).
- `03-videos/` (навчальні ролики).
- `04-diagrams/` (технічні схеми).
- `05-training/` (плани навчання та картки оцінювання).
- `06-external-resources/` (контакти постачальників, каталоги).
- `00-INDEX-GENERAL.md` (головний зміст та навігація).

## Найкращі практики

1. **Документуйте поступово** — одна процедура на місяць дасть чудовий результат за рік.
2. **Залучайте учня** — нехай він пише чернетки після спостереження, а ви лише виправляйте.
3. **Використовуйте відео** — смартфон та 3 хвилини коментарів під час роботи цінніші за сторінку тексту.
4. **Регулярно оновлюйте** — переглядайте базу раз на рік або при зміні нормативів.
5. **Створюйте стимули** — відзначайте прогрес учня та давайте йому більше автономії в міру навчання.
