# Воркфлоу: Замовлення Постачальнику

> **Орієнтовний час**: 15 хвилин на замовлення
> **Складність**: Середній
> **Категорія**: 🔧 Виробництво
> **Професії**: Торговці, Підрядники, Власники бізнесу

🌐 **Мови**: [English](supplier-order.en.md) | [Français](supplier-order.md) | [Українська 🇺🇦](supplier-order.uk.md)

---

## Кейс використання

Вам потрібно закупити матеріали або товари для проєкту чи складу. Цей воркфлоу допоможе вам:

- Сформувати точний перелік потреб (заявку).
- Вибрати найкращого постачальника за ціною та термінами.
- Сформувати професійне замовлення (Purchase Order).
- Перевірити відповідність поставки при отриманні.

> ⚠️ **Економія**: Оптимізація замовлень дозволяє зменшити витрати на 10-15% за рахунок оптових знижок та економії на логістиці.

## Попередні вимоги

- [ ] Cowork увімкнено в Claude Desktop.

- [ ] Контакти та прайс-листи основних постачальників.
- [ ] Створена робоча папка.

## Покрокова інструкція

### Крок 1: Консолідація потреб

```bash
mkdir -p ~/Cowork-Workspace/purchasing/{orders,deliveries}
```

Зберіть усі запити в один список:

```
Склади зведену заявку на закупівлю для: [Проєкт/Тиждень]

ПОТРЕБИ:
- З інвентарю: Кабель (200м), Розетки (50шт).
- З плану проєкту: Гіпсокартон (30 листів), Шпаклівка (10 мішків).

Пріоритет: КРИТИЧНО (потрібно до середи).
Збережи у: ~/Cowork-Workspace/purchasing/needs-[дата].xlsx
```

### Крок 2: Вибір та порівняння постачальників

```
Порівняй пропозиції для заявки [Дата]:

ПОСТАЧАЛЬНИК 1: "БудМаркет"
- Сума: 12 500 грн.
- Доставка: Завтра, 500 грн.
- Оплата: Рахунок з ПДВ.

ПОСТАЧАЛЬНИК 2: "ОптТорг"
- Сума: 11 800 грн.
- Доставка: Через 3 дні, безкоштовно.
- Оплата: Готівка або карта.

РЕКОМЕНДАЦІЯ: Оскільки термін критичний, обираємо Постачальника 1, незважаючи на вищу ціну.
```

### Крок 3: Генерація замовлення (Purchase Order)

```
Згенеруй замовлення № [Номер] для постачальника [Назва]:

ДЕТАЛІ ЗАМОВЛЕННЯ:
1. [Товар А] - [Кількість] - [Ціна].
2. [Товар Б] - [Кількість] - [Ціна].

УМОВИ ДОСТАВКИ:
Адреса: [Ваша адреса/Об'єкт].
Контакт: [Ім'я, телефон].
Бажаний час: [Вікно доставки].

УМОВИ ОПЛАТИ: Рахунок-фактура (чекаємо на імейл).

Збережи у: ~/Cowork-Workspace/purchasing/orders/PO-[номер].pdf
```

### Крок 4: Контроль отримання (Delivery Verification)

Коли машина приїхала, перевірте все за списком:

```
Проведи перевірку поставки за замовленням PO-[номер]:

ЧЕК-ЛИСТ ПРИЙМАННЯ:
- Кількість відповідає накладній? [Так/Ні]
- Цілісність пакування? [Так/Ні]
- Наявність сертифікатів якості? [Так/Ні]
- Наявність видаткової накладної? [Так/Ні]

ЗАУВАЖЕННЯ:
Виявлено 1 мішок порваної шпаклівки. Зазначено в накладній для повернення коштів.

Збережи звіт про отримання у: ~/Cowork-Workspace/purchasing/deliveries/receipt-[номер].pdf
```

### Крок 5: Передача в бухгалтерію

```
Підготуй пакет документів для оплати/обліку:
- Замовлення (PO).
- Фото видаткової накладної.
- Фото чека/рахунка.

Згрупуй у архів та надішли бухгалтеру.
```

## Найкращі практики

1. **Умови оплати** — Завжди уточнюйте, чи працює постачальник з ПДВ. Для платників ПДВ це економія 20%.
2. **Мінімальне замовлення** — Об'єднуйте дрібні замовлення, щоб отримати безкоштовну доставку.
3. **Резервні постачальники** — Майте контакти мінімум 2-3 баз, щоб не залежати від наявності товару в одного продавця.
4. **Перевірка при водієві** — Ніколи не відпускайте водія, не відкривши коробки з крихким товаром (плитка, санфаянс). Після підпису накладної претензії зазвичай не приймаються.
5. **Архівування** — Зберігайте всі накладні в цифровому вигляді (фото/скан) у папці проєкту. Паперові документи часто губляться на будівництві.
