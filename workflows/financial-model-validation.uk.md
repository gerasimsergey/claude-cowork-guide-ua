# Воркфлоу: Валідація фінансових моделей

🌐 **Мови**: [English](financial-model-validation.en.md) | [Français](financial-model-validation.md) | [Українська 🇺🇦](financial-model-validation.uk.md)

> **Орієнтовний час**: 30-45 хвилин
> **Складність**: Середня
> **Категорія**: 📊 Адміністративні
> **Професії**: Фінансові менеджери, Контролери, Власники бізнесу

---

## Кейс використання

У вас є фінансова модель (бюджет, прогноз, калькулятор ціноутворення) в Excel, і вам потрібно:

- Перевірити цілісність формул на всіх аркушах
- Виявити циклічні посилання або помилки
- Підтвердити логіку розрахунків
- Задокументувати припущення та залежності
- Створити аудиторський слід для зацікавлених сторін

> ⚠️ **Важливо**: Цей воркфлоу використовує **можливості Cowork з генерації Excel** (створення нових файлів аналізу). Він НЕ використовує **надбудову Claude для Excel** (надбудова Microsoft Excel для допомоги з формулами). Див. [порівняння](../reference/comparison.en.md#common-confusion).

## Покрокова інструкція

### Крок 1: Підготовка моделі до перевірки

```bash
# Створення структури робочого простору
mkdir -p ~/Cowork-Workspace/{input/financial-model,output}

# Копіювання моделі до робочого простору
cp ~/path/to/your-model.xlsx ~/Cowork-Workspace/input/financial-model/
```

**Порада**: Зробіть резервну копію оригінального файлу перед початком валідації.

### Крок 2: Аналіз загальної структури

Запит на отримання високорівневого огляду:

```
Проаналізуй Excel-файл у ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Надай:
1. Список аркушів із призначенням кожного
2. Ключові клітинки введення (де вказуються припущення)
3. Ключові клітинки виведення (фінальні результати)
4. Залежності між аркушами

Збережи резюме до: ~/Cowork-Workspace/output/model-structure.md
```

### Крок 3: Аудит формул

Перевірка цілісності формул:

```
Проведи аудит усіх формул у ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Створи звіт Excel у ~/Cowork-Workspace/output/formula-audit.xlsx з такими даними:

Sheet 1 - "Formula Inventory" (Інвентаризація формул):
- Посилання на клітинку
- Формула
- Назва аркуша
- Тип формули (SUM, IF, VLOOKUP тощо)
- Посилання (від яких клітинок залежить)

Sheet 2 - "Potential Issues" (Потенційні проблеми):
- Посилання на клітинку
- Тип проблеми (жорстко прописане значення у формулі, відсутність обробки помилок, складна вкладена формула)
- Пріоритет (Низький/Середній/Високий)
- Рекомендація

Sheet 3 - "Cross-Sheet Dependencies" (Залежності між аркушами):
- Аркуш-джерело
- Цільовий аркуш
- Кількість посилань
- Ключові пов'язані клітинки
```

### Крок 4: Виявлення помилок

Ідентифікація помилок у розрахунках:

```
Перевір ~/Cowork-Workspace/input/financial-model/your-model.xlsx на наявність:

1. Клітинок із помилками #REF!, #VALUE!, #DIV/0!, #N/A
2. Циклічних посилань
3. Прихованих клітинок із формулами
4. Неузгоджених шаблонів формул (напр., рядок 5 використовує іншу логіку, ніж рядки 1-4)

Збережи результати до: ~/Cowork-Workspace/output/error-report.md
```

### Крок 5: Документування припущень

Вилучення та документування припущень:

```
Переглянь ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Створи ~/Cowork-Workspace/output/assumptions.xlsx з такими даними:

Sheet 1 - "Input Assumptions" (Вхідні припущення):
- Назва параметра
- Поточне значення
- Одиниця виміру
- Джерело/Обґрунтування
- Дата останнього оновлення
- Чутливість (Низький/Середній/Високий вплив на результати)

Sheet 2 - "Calculated Assumptions" (Розрахункові припущення):
- Назва параметра
- Формула
- Поточне значення
- Залежності
```

### Крок 6: Перевірка чутливості

Тестування чутливості припущень:

```
Для фінансової моделі у ~/Cowork-Workspace/input/financial-model/your-model.xlsx:

Визнач 5 найбільш критичних вхідних припущень (тих, що мають найбільший вплив на кінцеві результати).

Створи ~/Cowork-Workspace/output/sensitivity-analysis.md, де задокументуй:
- Які припущення спричиняють найбільші відхилення
- Рекомендовані діапазони для планування сценаріїв
- "Червоні прапорці" у разі зміни припущень
```

### Крок 7: Генерація звіту про валідацію

Зведення результатів у резюме для керівництва:

```
Створи документ Word у ~/Cowork-Workspace/output/validation-report.docx

Включи:
1. Executive Summary (Резюме)
   - Мета моделі
   - Дата валідації
   - Загальна оцінка стану (Пройдено/Попередження/Невдача)

2. Результати
   - Знайдені проблеми з формулами
   - Виявлені помилки
   - Прогалини в припущеннях

3. Рекомендації
   - Необхідні критичні виправлення
   - Покращення відповідно до найкращих практик
   - Потреби в документації

4. Додатки
   - Детальний аудит формул (посилання на Excel-файл)
   - Список припущень (посилання на Excel-файл)
```

## Критерії успіху

✅ **Завершено, коли**:
- Усі формули задокументовані та перевірені
- Усі критичні помилки (#REF!, циклічні посилання) опрацьовані
- Припущення задокументовані із зазначенням джерел
- Звіт про валідацію передано зацікавленим сторонам

---

## Поширені труднощі

| Проблема | Рішення |
|----------|---------|
| Модель занадто велика (>10 МБ) | Розбийте на модулі, валідуйте окремо |
| Складні макроси VBA | Cowork не може виконувати макроси — задокументуйте їх вручну |
| Зовнішні посилання на дані | Зазначте залежності, валідуйте джерела даних окремо |
| Захищені паролем аркуші | Розблокуйте аркуші перед початком аналізу |

## Адаптації до регіонального контексту

Для підприємств у специфічних юрисдикціях (наприклад, європейських):

**Термінологія**:
- Використовуйте локалізовані назви функцій Excel: напр., `SOMME` замість `SUM`, `SI` замість `IF`
- Валюта: EUR (€) замість USD ($)
- Формат дати: ДД/ММ/РРРР (європейський) замість ММ/ДД/РРРР (американський)

**Відповідність**:
- Документуйте припущення для перевірки бухгалтером/аудитором
- Ведіть аудиторський слід для податкових перевірок (contrôle fiscal)
- Посилайтеся на відповідні стандарти бухгалтерського обліку

**Регіональні налаштування**:
```
При аналізі використовуй європейський синтаксис Excel:
- Роздільник у формулах: крапка з комою (;) замість коми (,)
- Десятковий роздільник: кома (,) замість крапки (.)
- Приклад: =SOMME(A1;A2) замість =SUM(A1,A2)
```

## Просунуті поради

**Великі моделі**:
- Обробляйте пакетами: валідуйте по одному аркушу за раз
- Зосередьтеся спочатку на критичних шляхах розрахунків
- Документуйте межі модулів для чіткості

**Регулярна валідація**:
- Зберігайте промпти для повторного використання щокварталу/щороку
- Створіть чек-лист для послідовної валідації
- Відстежуйте зміни між циклами валідації

**Комунікація із зацікавленими сторонами**:
- Спершу генеруйте резюме (1 сторінка)
- Надавайте детальний технічний додаток окремо
- Використовуйте візуальні засоби (діаграми) для відображення чутливості припущень

## Приклад промпту (повний воркфлоу)

```
У мене є фінансова модель у ~/Cowork-Workspace/input/financial-model/budget-2026.xlsx

Виконай цей воркфлоу валідації:

1. Аналіз структури:
   - Перелічи всі аркуші з їх призначенням
   - Визнач аркуші введення, розрахунків та виведення
   - Збережи до structure-analysis.md

2. Аудит формул:
   - Проведи інвентаризацію всіх формул
   - Познач потенційні проблеми (жорстко вписані значення, складна вкладеність)
   - Створи formula-audit.xlsx

3. Виявлення помилок:
   - Знайди помилки #REF!, #VALUE!, #DIV/0!
   - Перевір наявність циклічних посилань
   - Повідом про приховані формули
   - Збережи до error-report.md

4. Документування припущень:
   - Витягни всі вхідні припущення
   - Задокументуй джерела та чутливість
   - Створи assumptions.xlsx

5. Звіт про валідацію:
   - Резюме (1 сторінка)
   - Детальні результати
   - Рекомендації
   - Збережи до validation-report.docx

Використовуй європейський синтаксис Excel (роздільники — крапка з комою, локалізовані назви функцій).
```
