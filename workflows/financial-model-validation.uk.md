# Воркфлоу: Валідація Фінансових Моделей

🌐 **Мови**: [English](financial-model-validation.en.md) | [Français](financial-model-validation.md) | [Українська 🇺🇦](financial-model-validation.uk.md)

> **Орієнтовний час**: 30-45 хвилин
> **Складність**: Середній
> **Категорія**: 📊 Адміністративні
> **Професії**: Фінансові менеджери, Контролери, Власники бізнесу

---

## Кейс використання

У вас є фінансова модель (бюджет, прогноз, калькулятор цін) в Excel, і вам потрібно:

- Перевірити цілісність формул на всіх листах.
- Знайти циклічні посилання або помилки розрахунків.
- Підтвердити логіку обчислень.
- Задокументувати припущення та залежності.
- Створити звіт про аудит для зацікавлених сторін.

> ⚠️ **Важливо**: Цей воркфлоу використовує **можливості Cowork з генерації Excel** (створення нових аналітичних файлів). Він НЕ використовує **надбудову Claude в Excel** (Claude in Excel add-in). Див. [порівняння](../reference/comparison.uk.md#common-confusion).

---

## Попередні вимоги

- [ ] Cowork увімкнено (підписка Pro або Max).
- [ ] Файл фінансової моделі (.xlsx) підготовлений.
- [ ] Створена робоча папка.
- [ ] Базове розуміння структури вашої моделі.

## Покрокова інструкція

### Крок 1: Підготуйте модель для огляду

```bash
# Створіть структуру робочого простору
mkdir -p ~/Cowork-Workspace/{input/financial-model,output}

# Скопіюйте вашу модель у робочий простір
cp ~/шлях/до/вашої-моделі.xlsx ~/Cowork-Workspace/input/financial-model/
```

**Порада**: Завжди робіть резервну копію оригінального файлу перед початком валідації.

### Крок 2: Початковий аналіз структури

Запитайте огляд високого рівня:

```
Проаналізуй Excel файл за шляхом ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Надай:
1. Список листів та призначення кожного.
2. Ключові вхідні клітинки (де вводяться припущення/дані).
3. Ключові вихідні клітинки (фінальні результати/KPI).
4. Залежності між листами.

Збережи резюме у: ~/Cowork-Workspace/output/model-structure.md
```

### Крок 3: Аудит формул

Перевірка цілісності формул:

```
Проведи аудит усіх формул у ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Створи звіт Excel у ~/Cowork-Workspace/output/formula-audit.xlsx із такими листами:

Лист 1 - "Інвентаризація формул":
- Посилання на клітинку
- Формула
- Назва листа
- Тип формули (SUM, IF, VLOOKUP тощо)
- Залежності (від яких клітинок залежить)

Лист 2 - "Потенційні проблеми":
- Посилання на клітинку
- Тип проблеми (жорстко прописане значення у формулі, відсутність обробки помилок, занадто складна вкладена формула)
- Критичність (Низька/Середня/Висока)
- Рекомендація з виправлення

Лист 3 - "Залежності між листами":
- Лист-джерело
- Лист-ціль
- Кількість посилань
```

### Крок 4: Пошук помилок

Виявлення помилок розрахунків:

```
Перевір ~/Cowork-Workspace/input/financial-model/your-model.xlsx на наявність:

1. Клітинок із помилками #REF!, #VALUE!, #DIV/0!, #N/A.
2. Циклічних посилань.
3. Прихованих клітинок із формулами.
4. Неузгоджених паттернів формул (наприклад, рядок 5 використовує іншу логіку, ніж рядки 1-4).

Збережи результати у: ~/Cowork-Workspace/output/error-report.md
```

### Крок 5: Документування припущень

Вилучення та документація вхідних даних:

```
Переглянь ~/Cowork-Workspace/input/financial-model/your-model.xlsx

Створи ~/Cowork-Workspace/output/assumptions.xlsx:
- Назва параметра
- Поточне значення
- Одиниця виміру
- Джерело/Обґрунтування
- Чутливість (Низький/Середній/Високий вплив на результат)
```

### Крок 6: Генерація звіту про валідацію

Підготуйте фінальний документ для керівництва:

```
Створи документ Word за шляхом ~/Cowork-Workspace/output/validation-report.docx

Включи:
1. Executive Summary (Загальний висновок: Модель валідна / Потребує виправлення).
2. Виявлені проблеми з формулами та помилки.
3. Аналіз припущень.
4. Рекомендації щодо покращення та документації.
```

## Регіональні налаштування (Україна/ЄС)

Для коректної роботи з Excel в українському контексті використовуйте такі уточнення в промптах:

```
При аналізі та генерації Excel використовуй європейський синтаксис:
- Розділювач у формулах: крапка з комою (;) замість коми (,).
- Розділювач десяткових знаків: кома (,) замість крапки (.).
- Валюта: ГРН (UAH) або EUR (€).
- Формат дати: ДД.ММ.РРРР.
- Приклад: =SUM(A1;A2) замість =SUM(A1,A2).
```

## Поширені проблеми

| Проблема | Рішення |
|-------|----------|
| Файл занадто великий (>10МБ) | Розбийте на модулі, валідуйте листи окремо. |
| Складні макроси VBA | Cowork не може виконувати макроси — задокументуйте їх логіку вручну. |
| Пароль на листах | Зніміть захист перед аналізом. |

---

## Найкращі практики

1. **Версійність** — Завжди додавайте дату та версію до назви файлу після валідації.
2. **Чутливість** — Перевіряйте, як зміна ключового показника (наприклад, курсу валют або вартості сировини) впливає на фінальний прибуток.
3. **Зовнішні посилання** — Уникайте посилань на інші файли; краще копіюйте дані в саму модель для надійності.
4. **Прозорість** — Уникайте "магічних чисел" (жорстко прописаних значень) всередині формул. Виносьте їх в окремий блок припущень.
