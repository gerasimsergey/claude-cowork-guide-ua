# Воркфлоу: Щоденний Журнал Робіт

> **Орієнтовний час**: 10 хвилин на день
> **Складність**: Початківець
> **Категорія**: 🔧 Виробництво
> **Професії**: Підрядники, Виконроби, Майстри

🌐 **Мови**: [English](work-log.en.md) | [Français](work-log.md) | [Українська 🇺🇦](work-log.uk.md)

---

## Кейс використання

Вам потрібно документувати щоденну активність на будівельному майданчику або об'єкті надання послуг. Це критично для юридичного захисту, звітності перед замовником та контролю термінів. Цей воркфлоу допоможе вам:

- Створювати щоденні звіти про виконані роботи.
- Фіксувати погодні умови та простої (важливо для обґрунтування затримок).
- Документувати інциденти та зміни в проєкті.
- Вести систематичну фотофіксацію прогресу.

> ⚠️ **Юридична цінність**: Журнал робіт є доказом у разі суперечок. Дати, фото та підписи підтверджують факт виконання робіт у конкретний час.

## Покрокова інструкція

### Крок 1: Створення структури журналу

```bash
mkdir -p ~/Cowork-Workspace/projects/[назва-проєкту]/work-logs
mkdir -p ~/Cowork-Workspace/projects/[назва-проєкту]/photos
```

### Крок 2: Щоденний запис у журнал

Наприкінці кожного дня надайте Claude наступні дані:

```
Створи запис у журнал робіт за [Дата]:

ПРОЄКТ: [Назва/Адреса]
ПОГОДА: [Наприклад: Хмарно, +5°C. Опади з 10:00 до 12:00].
ПЕРСОНАЛ: [Ваша бригада: 4 чол. Підрядники: Електрики 2 чол].

ВИКОНАНІ РОБОТИ:
1. Заливка армопоясу (секція А).
2. Монтаж кабелю (2-й поверх) - 150 м.
3. Доставка вікон (прийнято 8 шт, без пошкоджень).

ІНЦИДЕНТИ/ПРОСТОЇ:
- Зупинка робіт на 2 години через дощ (бетонування перенесено).
- Зміна в плані: Замовник погодив перенесення розетки в кухні (+1 шт).

ФОТО: [Список назв файлів або опис того, що на фото].

Збережи запис у: ~/Cowork-Workspace/projects/[назва]/work-logs/2026-01-20.pdf
```

### Крок 3: Систематизація фотофіксації

Claude допоможе організувати фото:

```
Проаналізуй фото з папки ~/Cowork-Workspace/projects/[назва]/photos/today/
та перейменуй їх за стандартом:
РРРР-ММ-ДД_ЧЧММ_Опис.jpg

Приклад: 2026-01-20_1430_Армування_вузол_1.jpg
Сформуй звіт з фотофіксації (Photo Report) для замовника.
```

### Крок 4: Щотижневе зведення (Weekly Summary)

```
На основі щоденних логів за останній тиждень згенеруй звіт:
1. Загальний прогрес у % від плану.
2. Сумарні людино-години.
3. Список усіх виявлених проблем та їх статус.
4. План на наступний тиждень.

Збережи у: ~/Cowork-Workspace/projects/[назва]/work-logs/weekly-summary-week-[№].pdf
```

### Крок 5: Підсумковий звіт по завершенню етапу

```
Згенеруй фінальний звіт по етапу [Назва етапу]:
- Дати початку/завершення.
- Ключові досягнення.
- Акти прихованих робіт (посилання на фото).
- Відповідність кошторису.

Формат: PDF для підпису замовником.
```

## Найкращі практики

1. **Фіксуйте простій** — Якщо ви не можете працювати через відключення світла, відсутність матеріалів замовника або погоду — записуйте це щодня. Це захистить вас від штрафів за затримку.
2. **Акти прихованих робіт** — Робіть 5-10 фото кожного вузла, який буде закритий (труби в стяжці, проводка в штробах) з лінійкою для масштабу.
3. **Об'єктивність** — Пишіть факти, а не емоції. Замість "постачальник підвів" пишіть "матеріали не доставлені у визначений термін (10:00), доставка відбулася о 17:00".
4. **Хмарна синхронізація** — Налаштуйте автозавантаження фото в хмару (Google Drive/iCloud), щоб не втратити дані при пошкодженні смартфона на будівництві.
5. **Ділова мова** — Використовуйте стандартні терміни (КВЕДи, ДБН, ДСТУ), щоб звіт виглядав професійно перед корпоративними клієнтами.
