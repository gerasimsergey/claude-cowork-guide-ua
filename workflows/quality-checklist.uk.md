# Воркфлоу: Контроль Якості та Чек-листи

> **Орієнтовний час**: 15 хвилин на перевірку
> **Складність**: Початківець
> **Категорія**: 🔧 Виробництво
> **Професії**: Підрядники, Контролери якості, Виконроби

🌐 **Мови**: [English](quality-checklist.en.md) | [Français](quality-checklist.md) | [Українська 🇺🇦](quality-checklist.uk.md)

## Кейс використання

Вам потрібно гарантувати стабільну якість робіт на об'єкті та переконатися, що жодна важлива деталь не пропущена перед здачею етапу замовнику. Цей воркфлоу допоможе вам:

- Створювати стандартизовані чек-листи для різних видів робіт (електрика, сантехніка, оздоблення).
- Проводити систематичні інспекції згідно з ДБН та ДСТУ.
- Фіксувати дефекти та відстежувати їх виправлення.
- Формувати звіти про відповідність якості для замовника.

> ⚠️ **Ефект**: Систематичні перевірки зменшують кількість дефектів на 40-60% та знижують кількість скарг замовників удвічі.

## Покрокова інструкція

### Крок 1: Створення шаблону чек-листа

```bash
mkdir -p ~/Cowork-Workspace/quality/checklists
mkdir -p ~/Cowork-Workspace/quality/inspections
```

Попросіть Claude згенерувати чек-лист на основі норм:

```
Створи чек-лист контролю якості для: [Вид робіт, наприклад: Монтаж гіпсокартону]

ВИМОГИ:
- Базуйся на стандартах ДБН та ДСТУ.
- Кожен пункт має містити критерій "Пройдено/Не пройдено".
- Додай поле для фотодоказу.

СТРУКТУРА (приклад для оздоблення):
1. Площинність стін (допуск +/- 2мм на 2м).
2. Відсутність тріщин та сколів.
3. Обробка швів (наявність армувальної стрічки).
4. Кути (вертикальність та геометрія).
5. Чистота підрозетників.

Збережи шаблон у: ~/Cowork-Workspace/quality/checklists/finish-works-template.pdf
```

### Крок 2: Проведення інспекції

Під час обходу об'єкта надавайте Claude дані про виявлені факти:

```
Проведи інспекцію об'єкта [Назва] за шаблоном [Назва шаблону]:

РЕЗУЛЬТАТИ ПЕРЕВІРКИ:
1. Стіни в вітальні: Пройдено. Фото: IMG_01.jpg
2. Стіни в кухні: НЕ ПРОЙДЕНО. Виявлено відхилення 5мм на куті. Фото: IMG_02.jpg
3. Шви: Пройдено.
...

Згенеруй звіт про інспекцію (Inspection Report).
Збережи у: ~/Cowork-Workspace/quality/inspections/report-[дата].pdf
```

### Крок 3: Реєстрація та усунення недоліків (Punch List)

```
Створи список зауважень (Punch List) на основі останньої інспекції:

- Дефект: Відхилення площини в кухні.
- Відповідальний: Бригада оздоблювальників.
- Термін усунення: [Дата].
- Метод виправлення: Перешпаклювати кут.

Додай статус: "Очікує виправлення".
```

### Крок 4: Перевірка виправлень (Re-inspection)

```
Онови статус Punch List для об'єкта [Назва]:
- Дефект "Відхилення в кухні" — ВИПРАВЛЕНО.
- Додай фото-підтвердження: IMG_05_fixed.jpg.
- Дата перевірки: [Сьогодні].

Сформуй фінальний сертифікат якості етапу.
```

### Крок 5: Аналіз типових помилок

```
Проаналізуй останні 5 інспекцій і скажи:
- Які 3 дефекти зустрічаються найчастіше?
- Яка бригада робить найбільше помилок?
- Які рекомендації надати майстрам для запобігання цим помилкам у майбутньому?
```

## Найкращі практики

1. **Об'єктивність** — Використовуйте інструменти (лазерний рівень, правило, вологомір). Замість "стіна рівна" пишіть "відхилення 1мм на 2м".
2. **Фото до і після** — Завжди фіксуйте дефект і результат його виправлення з одного і того ж ракурсу.
3. **Критичні точки** — Визначте "червоні пункти", без проходження яких наступний етап робіт категорично не може бути розпочатий (наприклад: гідроізоляція санвузла).
4. **Залучення замовника** — Надсилайте проміжні звіти з чек-листами замовнику. Це знімає 90% питань під час фінальної здачі.
5. **Нормативна база** — Тримайте актуальні витяги з ДБН у папці `quality/standards` для швидкої аргументації перед підрядниками.
