# Воркфлоу: Створення Презентацій для Клієнтів

> **Орієнтовний час**: 30 хвилин
> **Складність**: Середній
> **Категорія**: 💼 Комерційні
> **Професії**: Власники бізнесу, Фрілансери, Менеджери з продажу

🌐 **Мови**: [English](presentation-slides.en.md) | [Français](presentation-slides.md) | [Українська 🇺🇦](presentation-slides.uk.md)

## Кейс використання

Вам потрібно представити свої послуги, проєкт або комерційну пропозицію клієнту. Цей воркфлоу допоможе вам:

- Скласти переконливу структуру презентації (логіка пітчу).
- Підготувати контент для професійних слайдів.
- Адаптувати аргументи під конкретні потреби клієнта.
- Підготувати скрипт виступу та відповіді на заперечення.

> ⚠️ **Ефект**: Структурована презентація підвищує конверсію в угоду на 30-50% порівняно з імпровізованою розмовою.

## Покрокова інструкція

### Крок 1: Аналіз контексту та бриф

```bash
mkdir -p ~/Cowork-Workspace/presentations/[назва-клієнта]
```

Сформуйте основу для Claude:

```
Підготуй бриф для презентації клієнту: [Назва клієнта]

ЦІЛЬ: [Напр: Підписати договір на річне обслуговування]
АУДИТОРІЯ: [Власник бізнесу / Технічний директор / Приватна особа]
ТРИВАЛІСТЬ: [Напр: 15 хвилин]

ПРОБЛЕМА КЛІЄНТА: [Що саме у них болить? Напр: Високі витрати на електроенергію]
НАШЕ РІШЕННЯ: [Головна перевага. Напр: Енергоаудит та модернізація зі швидкою окупністю]

Збережи бриф у: ~/Cowork-Workspace/presentations/[клієнт]/brief.txt
```

### Крок 2: Структурування слайдів

Попросіть Claude розписати структуру (по 1 ідеї на слайд):

```
Склади структуру презентації на 10-12 слайдів:

СЛАЙД 1: Титульний (Логотип, Назва проєкту).
СЛАЙД 2: Порядок денний (Про що поговоримо).
СЛАЙД 3: Ваша ситуація (Демонстрація розуміння потреб клієнта).
СЛАЙД 4: Наслідки бездіяльності (Чому проблему треба вирішувати зараз).
СЛАЙД 5: Наша пропозиція (Суть рішення).
СЛАЙД 6: Як це працює (Процес у 3 кроки).
СЛАЙД 7: Кейси та досвід (Наші успішні проєкти).
СЛАЙД 8: Команда (Хто буде працювати над проєктом).
СЛАЙД 9: Вартість та пакети (Варіанти співпраці).
СЛАЙД 10: Гарантії та терміни.
СЛАЙД 11: Наступні кроки (Що робимо завтра).
СЛАЙД 12: Дякуємо за увагу / Контакти.

Збережи структуру у: ~/Cowork-Workspace/presentations/[клієнт]/structure.txt
```

### Крок 3: Наповнення контентом (Copywriting)

```
Напиши тексти для слайдів 3, 5 та 9:
- Використовуй тезу: "Менше слів — більше сенсу".
- Додай 3 ключові буліти для кожного слайда.
- Виділи головну цифру (напр: "Заощадження 20%").
```

### Крок 4: Візуальні рекомендації

```
Запропонуй візуальний стиль для цієї презентації:
- Які кольори використати для [Галузь]?
- Які фото підібрати для ілюстрації етапу монтажу?
- Який тип шрифту виглядає сучасно та професійно?
```

### Крок 5: Підготовка до виступу та заперечень

```
Підготуй список потенційних заперечень клієнта:
1. "Це дорого" → [Варіант відповіді з акцентом на окупність].
2. "Нам треба подумати" → [Варіант відповіді для стимулювання рішення].
3. "У конкурентів дешевше" → [Варіант відповіді з акцентом на якість/гарантію].

Напиши скрипт вступного слова на 1 хвилину.
```

## Найкращі практики

1. **Правило 10/20/30** — 10 слайдів, 20 хвилин виступу, 30-й шрифт (мінімум). Слайди мають бути читабельними з останнього ряду.
2. **Одна думка — один слайд** — Не намагайтеся запхати весь прайс-лист на один слайд. Клієнт перестане вас слухати, намагаючись прочитати дрібний шрифт.
3. **Більше візуалу** — Використовуйте схеми, графіки та реальні фото. Текст — це лише опори для вашої розмови.
4. **Контрастність** — Темний текст на світлому фоні — це стандарт, який ніколи не підводить (особливо при поганому освітленні з проєктором).
5. **Заклик до дії в кінці** — Презентація без зрозумілого наступного кроку (запис на заміри, підписання договору) — це просто витрачений час.
