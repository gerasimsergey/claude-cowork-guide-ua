## Вибір моделі

Cowork підтримує різні моделі Claude. З моменту запуску Sonnet 4.6 (17 лютого 2026 року) критерії вибору моделі суттєво змінилися.

### Доступні моделі

| Модель | Найкраще для | Швидкість | Вікно контексту | Вартість |
|-------|----------|-------|----------------|------------|
| **Haiku** | Дуже прості завдання, швидкі запити | Дуже швидка | Стандартне | Низька |
| **Sonnet 4.6** ⭐ | Агентські завдання, автоматизація файлів, щоденні воркфлоу | Швидка | 1 млн токенів (бета), 128К вивід | Стандартна |
| **Opus 4.6** | Глибокі міркування, науковий аналіз, складні мульти-агентські завдання | Повільніша | 1 млн токенів (бета), 128К вивід | Вища (5x Sonnet) |

**Примітки до моделей** (Лютий 2026):
- **Sonnet 4.6** (випущена 17 лютого 2026): Тепер рекомендована модель за замовчуванням для Cowork. Досягає 72.5% на OSWorld-Verified (бенчмарк використання комп'ютера) проти 72.7% у Opus 4.6 — продуктивність в агентських завданнях майже ідентична при вартості у 5 разів меншій. Також включає вікно контексту 1 млн токенів (бета), 128К вивід та адаптивне мислення.
- **Opus 4.6** (випущена 5 лютого 2026): Зберігає переваги для глибоких наукових міркувань (91.3% на GPQA Diamond) та складної координації кількох агентів. Для стандартних операцій Cowork з файлами та автоматизації різниця в продуктивності з Sonnet 4.6 незначна.
- **Ущільнення контексту (бета)**: Cowork тепер використовує автоматичне ущільнення для стиснення історії розмов, що дозволяє проводити довші сесії без втрати важливого контексту.

### Коли яку використовувати

| Тип завдання | Рекомендована модель | Чому |
|-----------|-------------------|-----|
| Організація файлів, перейменування | **Sonnet 4.6** | Агентські завдання — сильна сторона Sonnet |
| Вилучення даних з чеків, OCR | **Sonnet 4.6** | Виклик інструментів — №1 рейтинг Sonnet |
| Чернетки листів, створення документів | **Sonnet 4.6** | Швидко, достатня якість |
| Щоденна автоматизація, завдання за розкладом | **Sonnet 4.6** | Та ж продуктивність, у 5 разів дешевше |
| Мульти-джерельний аналіз досліджень | **Sonnet 4.6** | Контекст 1 млн токенів для великих обсягів |
| Перегляд контрактів, юридичний аналіз | **Opus 4.6** | Перевага в глибоких міркуваннях |
| Складні наукові/технічні звіти | **Opus 4.6** | Потрібен рівень міркувань GPQA |
| Координація кількох агентів | **Opus 4.6** | Логіка координації краща в Opus |

### Поради щодо вибору

1. **Використовуйте Sonnet 4.6 за замовчуванням** — вона справляється з 90%+ завдань Cowork з продуктивністю, майже ідентичною до Opus.
2. **Перемикайтеся на Opus 4.6 тільки коли**:
   - Результати потребують експертного рівня міркувань (юридичні, наукові, регуляторні).
   - Завдання передбачає складну координацію кількох агентів.
   - Якість виводу Sonnet стабільно низька, незважаючи на якісні промпти.
3. **Уникайте Opus для файлових операцій** — немає відчутної переваги над Sonnet для організації файлів, вилучення даних або виконання запланованої автоматизації.
4. **Стежте за квотою** — Opus споживає в 5 разів більше квоти за токен. На плані Pro це швидко відчувається.

> **Користувачам Pro**: Sonnet 4.6 — ваш вибір для всього. Резервуйте Opus для рідкісних завдань, де глибина міркувань дійсно критична.
>
> **Користувачам Max**: Все одно починайте з Sonnet 4.6. Перемикайтеся на Opus для перегляду контрактів, стратегічного аналізу або коли виводу Sonnet бракує глибини.

## Детальний огляд можливостей

### 1. Організація файлів

Cowork чудово справляється з організацією великої кількості файлів:

```
INPUT:  500 файлів у папці Downloads
OUTPUT: Організована структура за типом/датою/проєктом
```

**Що робить Cowork**:
- Аналізує назви файлів, типи, контент.
- Пропонує схему організації.
- Створює структуру папок.
- Переміщує файли (після вашого схвалення).
- Генерує звіт про організацію.

**Обмеження**:
- Не завжди надійно отримує метадані (дата створення тощо).
- Не може читати контент деяких бінарних форматів.
- Покладається на назви файлів/розширення для категоризації.

### 2. Синтез документів

Об'єднання кількох джерел у структуровані результати:

```
INPUT:  15 нотаток із зустрічей, 3 звіти, 5 імейлів (як текстові файли)
OUTPUT: Виконавче резюме з ключовими рішеннями та завданнями
```

**Що робить Cowork**:
- Читає всі вихідні документи.
- Ідентифікує ключові теми та інформацію.
- Структурує в запитаний формат.
- Генерує фінальний документ.

**Обмеження**:
- Ліміти вікна контексту (~200К токенів).
- Немає прямого доступу до пошти чи календаря.
- Якість синтезу залежить від чіткості джерел.

### 3. Вилучення даних

Вилучення структурованих даних із неструктурованих джерел:

```
INPUT:  20 фотографій або скріншотів чеків
OUTPUT: Таблиця Excel з датою, продавцем, сумою, категорією
```

**Що робить Cowork**:
- OCR (оптичне розпізнавання) на зображеннях.
- Ідентифікує релевантні поля.
- Нормалізує формат даних.
- Створює Excel з формулами.

**Обмеження**:
- **Вилучення полів**: ~97% точність (продавець, дата, суми).
- **Вилучення окремих позицій**: ~63% точність (рядки таблиць) — перевіряйте вручну.
- Рукописний текст — це виклик.
- Деякі формати чеків можуть розпізнаватися некоректно.
- Плануйте 30-50% "зекономленого" часу на виправлення помилок.

> ⚠️ **Незалежних бенчмарків** для інструментів ШІ-організації файлів не існує. Твердження про продуктивність — це маркетинг виробників. Будьте реалістами.

### 4. Генерація звітів

Створення форматованих звітів із сирих даних:

```
INPUT:  CSV з даними про продажі
OUTPUT: Форматований звіт із діаграмами та аналізом
```

**Що робить Cowork**:
- Аналізує паттерни даних.
- Створює зведену статистику.
- Генерує візуальні представлення.
- Форматує для вказаного виводу.

**Обмеження**:
- Можливості діаграм у виводі Excel обмежені.
- Складні візуалізації потребують ручного доопрацювання.
- Статистичний аналіз є базовим.

### 5. Збір досліджень

Збір та організація досліджень із багатьох джерел:

```
INPUT:  Тема: "Інструменти продуктивності для віддаленої роботи"
OUTPUT: Документ дослідження з матрицею порівняння
```

**Що робить Cowork**:
- Використовує Chrome для веб-дослідження.
- Вилучає релевантну інформацію.
- Організовує в структурований формат.
- Наводить джерела.

**Обмеження**:
- Немає доступу до платного контенту (paywalls).
- Веб-дослідження повільніше, ніж прямий API.
- Якість залежить від доступної публічної інформації.

### Не плутайте з: Claude в Excel Add-in

> ⚠️ **Важливе уточнення**: Цей розділ описує **можливості Cowork з генерації Excel** (створення `.xlsx` файлів із даних). Це НЕ те саме, що **надбудова Claude в Excel** (Claude in Excel add-in), запущена 24 січня 2026 року.
>
> **Ключові відмінності**:

> - **Claude в Excel**: Допомагає з формулами/аналізом всередині вже відкритих файлів Excel.
>
> Див. [повне порівняння](../reference/comparison.uk.md#common-confusion).

### Регіональні особливості

Синтаксис формул Excel відрізняється за регіонами:
- **US/UK**: `=SUM(A1,A2)` (розділювач кома)
- **EU/Україна**: `=SUM(A1;A2)` (розділювач крапка з комою)

**Порада**: Вказуйте регіональні налаштування в промптах:
```
Створи файл Excel, використовуючи європейський синтаксис формул (розділювач крапка з комою).
```

## Використання вікна контексту

Cowork розділяє ліміт контексту Claude (~200К токенів), але **ефективна ємність становить ~165К**.

### Чому 165К, а не 200К?

Системні витрати споживають токени ще до початку завдання:
- Визначення інструментів: ~10К токенів.
- Інструкції з безпеки: ~10К токенів.
- Логи виконання: ~5-15К токенів.

**Ефективний ліміт**: ~165-175К корисних токенів.

### Практичні межі

| Тип контенту | Приблизна ємність |
|--------------|---------------------|
| Текстові сторінки | 100-400 сторінок |
| Документи | 40-80 типових документів |
| Рядки таблиць | 8,000-40,000 рядків |
| Зображення (OCR) | 40-80 зображень |

### Коли ліміт вичерпано

**Повідомлення про помилку**:
```
Context limit reached
```

**Симптоми**:
- Cowork зупиняється посеред завдання (часто біля 165К, не 200К).
- Результати неповні.
- Тихий збій без чіткого повідомлення.

**Рішення**:
- Розбивайте роботу на порції по 10-20 файлів.
- Обробляйте файли групами.
- Зберігайте проміжні результати в контрольні файли.
- Починайте нову розмову для нових завдань.

## Розширення та плагіни

Cowork підтримує офіційні розширення, які розширюють його можливості для спеціалізованих воркфлоу. Плагіни надаються Anthropic та інтегруються безпосередньо в інтерфейс Cowork.

### Доступні розширення

**Claude Legal** (анонсовано 3 лютого 2026):
- **Мета**: Автоматизація перегляду юридичних документів та виявлення ризиків.
- **Ключові можливості**:
  - Автоматизований аналіз контрактів та вилучення ключових умов.
  - Виявлення проблем із відповідністю та ризиками.
  - Тріаж (сортування) NDA та угод.
- **Використання для SMB**:
  - Перевірка контрактів перед підписанням.
  - Виявлення проблемних пунктів в угодах із постачальниками.
  - Створення чек-листів відповідності для специфічних секторів.

> ⚠️ **Юридичне застереження**: Claude Legal НЕ надає юридичних консультацій. Він допомагає в аналізі документів. Всі висновки мають бути перевірені кваліфікованим юристом.

**Приклад промпту**:
```
Review the contract in ~/Cowork-Workspace/contracts/vendor-agreement.pdf
Identify key terms, obligations, and potential risks.
Generate a summary with flagged issues for legal review.
```

#### Функціональні плагіни за відділами

Anthropic запустив готові плагіни, що об'єднують конектори для специфічних функцій:

#### Створіть власний плагін (без програмування)

5. Об'єднайте з релевантними конекторами.

## Нові можливості (Лютий 2026)

### Заплановані завдання (Scheduled Tasks)

Cowork може автоматизувати регулярні завдання, запускаючи їх у встановлений час.

#### Два типи завдань

1. **Повторювані (Recurring)** — запускаються автоматично з інтервалами (щогодини, щодня, щотижня).
2. **За запитом (On-demand)** — запускаються вручну одним кліком "Run now".

#### Як налаштувати

1. **Claude Desktop → ліва панель → Scheduled**.
2. **New Task**.
3. Напишіть промпт (рекомендовано формат CTOC).
4. Виберіть тип: Recurring (встановіть графік) або On-demand.
5. Активуйте.

> **Важливо**: Комп'ютер має бути ввімкнений, а Claude Desktop відкритий. Якщо ПК спить, завдання виконається після пробудження.

#### 4 основні паттерни автоматизації

**Паттерн 1: Щоденний ранковий брифінг** (щодня о 9:00)
```
CONTEXT: Файли, додані вчора в ~/Cowork-Workspace/input/daily/
TASK: Підсумуй нові документи, ключову інформацію та термінові справи на сьогодні.
OUTPUT: ~/Cowork-Workspace/output/brief-[date].md
```

**Паттерн 2: Щотижневий звіт про продажі** (щопонеділка о 8:00)
```
CONTEXT: Файли в ~/Cowork-Workspace/input/weekly/ за останні 7 днів.
TASK: Склади щотижневе резюме: підсумки, важливі моменти, відкриті питання.
OUTPUT: ~/Cowork-Workspace/output/weekly-[date].docx
```

## Пам'ять між сесіями (через Desktop Commander)

За замовчуванням кожна сесія Cowork починається з чистого аркуша. З **Desktop Commander** ви можете вирішити це за допомогою файлу `memory.md`.

1. Створіть `~/Cowork-Workspace/memory.md` з вашим контекстом.
2. На початку сесії скажіть: "Read ~/Cowork-Workspace/memory.md first. Then [ваш запит]".
