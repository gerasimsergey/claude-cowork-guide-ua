## Попередні вимоги

Перш ніж почати, переконайтеся, що у вас є:

| Вимога | Як перевірити |
|-------------|--------------|
| **Підписка Pro або Max** | claude.ai → Settings → Subscription показує "Pro" або "Max" |
| **macOS або Windows** | macOS: Apple menu → About This Mac; Windows: Settings → System → About |
| **Додаток Claude Desktop** | Папка Applications або пошук Spotlight |
| **Остання версія додатка** | Див. кроки перевірки нижче ⚠️ |

### ⚠️ Переконайтеся, що у вас остання версія Claude Desktop

**Важливо**: Cowork вимагає найновішої версії Claude Desktop. Якщо ви не бачите напису "Cowork" у селекторі режимів у верхній частині додатка, ваша версія застаріла.

**Як перевірити:**

1. Відкрийте додаток Claude Desktop.
2. Подивіться на верхню частину бічної панелі — ви маєте бачити три режими:
   ```
   Chat    Cowork    Code
   ```
3. Якщо ви **не бачите "Cowork"**, оновіть додаток:
   - Перейдіть у меню Claude Desktop → **Check for Updates**
   - Або завантажте останню версію з [claude.ai/download](https://claude.ai/download)

> **Офіційна документація**: Для додаткової допомоги див. [посібник Anthropic "Getting Started with Cowork"](https://support.claude.com/en/articles/13345190-getting-started-with-cowork)

### Рівні підписки

| Тариф | Вартість | Використання Cowork |
|------|------|--------------|
| **Pro** | $20/міс | Помірне використання (~1-1.5 год інтенсивної роботи до скидання) |

### Немає доступу?

| Ситуація | Дія |
|-----------|--------|
| Безкоштовний тариф | Перейдіть на Pro ($20) або Max ($100+) |
| Linux | Чекайте на розширення платформ (Linux поки не анонсовано) |

## Крок 1: Увімкніть Cowork

### 1.1 Відкрийте налаштування

1. Запустіть додаток **Claude Desktop**.
2. Натисніть на **іконку профілю** (верхній правий кут).
3. Виберіть **Settings**.

### 1.2 Активуйте функцію

1. Перейдіть до розділу **Features** або **Beta Features**.
2. Знайдіть перемикач **Cowork**.
3. Увімкніть його.

> **Примітка**: Точне розташування може дещо змінюватися з оновленнями додатка.

### 1.3 Перевірте активацію

Після увімкнення ви маєте побачити:
- Нову опцію "Cowork" у селекторі режимів розмови.
- Або окремий розділ/вкладку Cowork.

## Крок 2: Створіть свій робочий простір

**Важливо**: Ніколи не надавайте Cowork доступ безпосередньо до папок "Документи", "Робочий стіл" або домашньої папки користувача.

### 2.1 Створіть виділену папку

Відкрийте термінал і виконайте:

```bash
mkdir -p ~/Cowork-Workspace/{input,output}
```

Це створить структуру:
```
~/Cowork-Workspace/
├── input/    # Файли, які ви хочете обробити
└── output/   # Куди Cowork покладе результати
```

### 2.2 Надайте доступ до папки

1. Почніть нову розмову в режимі Cowork.
2. Коли з'явиться запит на доступ до папки, виберіть `~/Cowork-Workspace/`.
3. Надайте доступ **тільки** до цієї папки.

### 2.3 Перевірте доступ

Запитайте у Cowork:
```
List the contents of my workspace folder
```

Очікувана відповідь: Показати папки `input/` та `output/`.

## Крок 3: Ваш перший воркфлоу

Давайте виконаємо простий, але повний робочий процес, щоб переконатися, що все працює.

### 3.1 Підготуйте тестові файли

Створіть кілька тестових файлів у папці input:

```bash
cd ~/Cowork-Workspace/input

# Створення зразків файлів
echo "Нотатки із зустрічі в понеділок" > meeting-monday.txt
echo "Нотатки із зустрічі в середу" > meeting-wednesday.txt
echo "Оновлення статусу проєкту" > project-status.txt
echo "Різні думки" > misc-notes.txt
```

### 3.2 Запустіть перше завдання

У Cowork введіть:

```
Organize the files in ~/Cowork-Workspace/input/ into subfolders
by category. Create a summary of what you organized in the output folder.
```

### 3.3 Що має статися

1. **Відображення плану**: Cowork покаже свої заплановані дії.
2. **Ваше схвалення**: Ви перевіряєте та схвалюєте план.
3. **Виконання**: Cowork реорганізує файли.
4. **Звіт**: Створюється резюме (summary) у папці output.

### 3.4 Перевірте результати

Перевірте результат:

```bash
ls -la ~/Cowork-Workspace/input/
ls -la ~/Cowork-Workspace/output/
```

Ви маєте побачити:
- Упорядковані підпапки в `input/`.
- Файл звіту в `output/`.

## Крок 4: Розуміння робочого процесу

### Цикл роботи Cowork

Кожне завдання в Cowork слідує цій схемі:

```
┌─────────────────────────────────────────────────────┐
│                   ВАШ ЗАПИТ                          │
│     "Упорядкуй мої файли за категоріями"            │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│                    АНАЛІЗ                            │
│     Cowork вивчає ваші файли                        │
│     Визначає закономірності та категорії            │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│               ПРОПОЗИЦІЯ ПЛАНУ                       │
│     "Я створю 3 папки і переміщу X файлів..."       │
│     ⚠️ ВИ ПЕРЕВІРЯЄТЕ ЦЕ ПЕРЕД ВИКОНАННЯМ            │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│                   ВАШЕ СХВАЛЕННЯ                     │
│     "Так, виконуй" або "Ні, зміни план"             │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│                   ВИКОНАННЯ                          │
│     Cowork виконує схвалені дії                     │
└─────────────────────┬───────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│                     ЗВІТ                             │
│     Резюме того, що було зроблено                   │
└─────────────────────────────────────────────────────┘
```

### Ключові моменти

1. **Завжди перевіряйте план** — це ваша точка контролю безпеки.
2. **Будьте конкретними** — розпливчасті запити призводять до неочікуваних результатів.
3. **Починайте з малого** — протестуйте на кількох файлах, перш ніж давати великі масиви.
4. **Перевіряйте результати** — переконайтеся у правильності виводу перед наступними кроками.

## Крок 5: Фреймворк CTOC

Досвідчені користувачі структурують кожен промпт для Cowork за чотирма компонентами:

```
CONTEXT → TASK → OUTPUT → CONSTRAINTS
(КОНТЕКСТ → ЗАВДАННЯ → РЕЗУЛЬТАТ → ОБМЕЖЕННЯ)
```

### Фреймворк

| Компонент | Що це таке | Приклад |
|-----------|------------|---------|
| **C**ontext | Фон, файли, ситуація | "У мене є 50 чеків із бізнес-поїздки до Парижа..." |
| **T**ask | Чітка одна ціль | "Витягни всі витрати в електронну таблицю" |
| **O**utput | Точний формат і місце | "Збережи як ~/Cowork-Workspace/output/paris-expenses.xlsx" |
| **C**onstraints | Правила, межі, вподобання | "Використовуй валюту EUR, формули з крапкою з комою, категорізуй за типом" |

### Приклад CTOC

```
CONTEXT: У мене є нотатки із зустрічей за останній місяць у папці ~/Cowork-Workspace/input/notes/.
Вони від різних членів команди з непослідовним форматуванням.

TASK: Створи зведений звіт про статус на основі цих нотаток.

OUTPUT: Збережи як ~/Cowork-Workspace/output/team-status-january.docx
з розділами: Стисле резюме, Прогрес за проєктами, Блокери, Наступні кроки.

CONSTRAINTS: Обсяг до 3 сторінок. Зосередься на пунктах, що потребують дій.
Виділи всі згадані ризики.
```

### Швидкі поради

| Порада | Приклад |
|---------|---------|
| **Будьте точними** | ✅ "файли в ~/Cowork-Workspace/input/" замість ❌ "мої файли" |
| **Вказуйте вивід** | ✅ "збережи в ~/output/report.docx" замість ❌ "створи звіт" |
| **Описуйте формат** | ✅ "колонки: Дата, Сума, Категорія" замість ❌ "зроби таблицю" |
| **Додавайте обмеження** | ✅ "використовуй європейський синтаксис формул" |

### Розбивайте складні завдання

Замість:
```
❌ "Оброби всі мої чеки, створи звіти про витрати та впорядкуй за місяцями"
```

Робіть так:
```
✅ Крок 1: "Перелічи всі файли чеків у папці ~/Cowork-Workspace/input/"
✅ Крок 2: "Витягни дані про витрати з цих чеків у єдиний файл Excel"
✅ Крок 3: "Додай у файл Excel листи зі щомісячними підсумками"
```

Такий підхід також оптимізує використання токенів (див. [Шпаргалку](../reference/cheatsheet.uk.md) щодо лімітів).

## Крок 6: Персоналізуйте свій профіль (Опціонально)

Створіть файл особистого профілю, щоб Cowork знав ваші вподобання та стиль спілкування.

### 6.1 Створіть файл профілю

У вашому робочому просторі створіть `my-profile.md`:

```markdown
# Мій комунікаційний профіль

## Тон та стиль
- Офіційний (з клієнтами на "ви") / Неформальний (на "ти")
- Перевага стислим пунктам (bullet points)
- Без жаргонізмів та англіцизмів

## Чого я НІКОЛИ не роблю
- Не використовую знаки оклику в імейлах
- Не обіцяю конкретних дедлайнів
- Не пропускаю привітання

## Підпис за замовчуванням
З повагою,
[Ваше Ім'я] - [Компанія]
```

### 6.2 Використовуйте свій профіль

Починайте кожну розмову з:
```
Read my-profile.md first. Then [ваш фактичний запит]
```

**Приклад**:
```
Read my-profile.md first. Then draft a follow-up email to a client
who hasn't responded to our quote in 2 weeks.
```

### 6.3 Переваги

| Перевага | Чому це важливо |
|---------|----------------|
| **Єдиний голос** | Всі результати відповідають вашому стилю |
| **Економія часу** | Не потрібно щоразу повторювати вподобання |
| **Узгодженість команди** | Поділіться цим файлом з колегами |
| **Портативність** | Цей же файл працює з іншими ШІ-інструментами |

> **Порада**: 80% ефективного профілю — це те, чого ви *не* хочете. Зосередьтеся на обмеженнях та антипатернах.

## Крок 7: Інтеграція з Chrome (Опціонально)

Cowork може використовувати Chrome для завдань веб-дослідження.

### Увімкніть доступ до Chrome

1. Коли Cowork попросить дозвіл на Chrome, уважно перегляньте запит.
2. Надавайте доступ лише для конкретних завдань дослідження.
3. Скасуйте доступ після завершення завдання.

### Приклад завдання веб-дослідження

```
Research the top 5 project management tools for small teams.
Save your findings to ~/Cowork-Workspace/output/pm-tools-research.md
with a comparison table.
```

### Примітка щодо безпеки

- Переглядайте кожну веб-дію, яку пропонує Cowork.
- Не дозволяйте Cowork заповнювати форми або здійснювати покупки.
- Скасовуйте доступ до Chrome, коли він не потрібен.

## Крок 8: Встановіть Desktop Commander (Рекомендовано)

Desktop Commander — це безкоштовне офіційне розширення, яке розширює можливості Cowork. Більшість користувачів отримують від нього користь, особливо для повторюваної роботи.

### Що дає Desktop Commander

| Можливість | Без Desktop Commander | З Desktop Commander |
|-----------|--------------------------|----------------------|
| **Доступ до файлів** | Тільки папка Workspace | Будь-яка папка, яку ви дозволите |
| **Пам'ять між сесіями** | Відсутня — кожен раз з нуля | Постійна через файл memory.md |
| **Налаштування MCP серверів** | Ручне, технічне | Встановлення в один клік |

### Встановлення

1. Відкрийте Claude Desktop → вкладка **Customize** (або Settings → Extensions).
2. Знайдіть **Desktop Commander** у списку.
3. Натисніть **Install**.
4. Перезапустіть Claude Desktop, коли з'явиться запит.

Займає менше 2 хвилин. Технічні знання не потрібні.

### Налаштуйте файл пам'яті

Після встановлення Desktop Commander створіть файл постійної пам'яті, щоб Cowork пам'ятав ваш контекст між сесіями:

1. Створіть `~/Cowork-Workspace/memory.md`.
2. Додайте свій контекст, ось шаблон для початку:

```markdown
# Моя пам'ять Cowork

## Бізнес-контекст
- Тип бізнесу: [ваш тип — консалтинг, роздрібна торгівля, послуги тощо]
- Основні формати файлів: [Word, Excel, PDF тощо]

## Вподобання клієнтів
- Клієнт Петренко: офіційний тон, віддає перевагу інвойсам у PDF
- Клієнт Іваненко: потребує деталізованих кошторисів з розподілом праці та матеріалів

## Чого я ніколи не роблю
- Не використовую знаки оклику в листах клієнтам
- Не обіцяю конкретних дат доставки без попередньої перевірки

## Повторювані завдання
- Щотижня: [що ви збираєте або звітуєте щотижня]
- Щомісяця: [що ви робите в кінці місяця]
```

3. Починайте майбутні сесії з: "Read ~/Cowork-Workspace/memory.md first. Then [ваш фактичний запит]"

### Чому це важливо

Без файлу пам'яті вам довелося б повторювати вподобання та контекст на початку кожної сесії. З ним Cowork продовжує з того місця, де зупинився, знаючи ваших клієнтів, ваш стиль і ваші регулярні робочі паттерни.

> **Примітка**: Desktop Commander доступний через вкладку Customize у Claude Desktop. Якщо ви не бачите його одразу, перевірте Settings → Extensions.

## Крок 9: Увімкніть Computer Use (macOS, Опціонально)

Computer Use дозволяє Claude керувати вашим робочим столом напряму — відкривати додатки, клікати, заповнювати форми, переміщатися в браузері. Доступно на планах Pro та Max, тільки для macOS.

> **Примітка**: Computer Use доступний на планах Pro та Max. Використовуйте його для завдань під наглядом, де ви можете перевіряти кожну дію.

### 9.1 Увімкніть у Claude Desktop

1. Відкрийте **Claude Desktop**.
2. Перейдіть до **Settings → Features → Computer Use**.
3. Перемкніть у положення **On**.
4. Підтвердьте діалогове вікно дозволів.

### 9.2 Надайте системні дозволи macOS

Claude потребує два системні дозволи для керування екраном:

**Screen Recording** (запис екрана — щоб бачити, що відбувається):
1. Відкрийте **System Settings → Privacy & Security → Screen Recording**.
2. Знайдіть **Claude Desktop** у списку.
3. Увімкніть (якщо додатка немає в списку, натисніть `+` і додайте вручну).

**Accessibility** (доступність — щоб клікати та друкувати):
1. Відкрийте **System Settings → Privacy & Security → Accessibility**.
2. Знайдіть **Claude Desktop** у списку.
3. Увімкніть.

### 9.3 (Опціонально) З'єднайте телефон через Dispatch

Щоб надсилати завдання на комп'ютер зі свого телефону, поки ви не на місці:

1. Відкрийте **Claude для iOS або Android**.
2. У гілці Cowork натисніть **Pair Desktop**.
3. Відскануйте **QR-код**, показаний у додатку Claude Desktop.
4. Ваш телефон підключено — надсилайте завдання віддалено.

> **Вимоги**: Комп'ютер має бути ввімкнений, а додаток Claude Desktop відкритий. Claude використовує з'єднану гілку (Dispatch) для виконання завдань, поки вас немає.

### Примітки щодо безпеки Computer Use

- Claude просить **явний дозвіл на кожне завдання** перед доступом до нового додатка.
- Anthropic рекомендує **не надавати доступ до конфіденційних додатків** (банкінг, здоров'я, юридичні документи).
- Claude навчений **відмовляти** у: торгівлі акціями, збереженні конфіденційних паролів, зборі біометричних даних.
- **Вартість токенів вища**, ніж у стандартного Cowork: кожен цикл дії потребує скріншота.

## Вирішення проблем першого запуску

### "Cannot access folder" (Не вдається отримати доступ до папки)

1. Перейдіть у System Settings → Privacy & Security → Files and Folders.
2. Знайдіть Claude Desktop.
3. Переконайтеся, що ваша робоча папка в списку та увімкнена.

### "Cowork option not visible" (Опція Cowork не відображається)

**Найчастіша причина**: Застаріла версія додатка.

1. **Перевірте версію** — див. розділ [Попередні вимоги](#️-переконайтеся-що-у-вас-остання-версія-claude-desktop) вище.
2. Якщо ви не бачите "Chat | Cowork | Code" зверху, оновіть додаток.
3. Після оновлення перевірте Settings → Features → переконайтеся, що Cowork увімкнено.
4. Перезапустіть додаток.

### "Plan seems wrong" (План здається помилковим)

1. **Не схвалюйте** план.
2. Напишіть: "Stop. Let me clarify: [ваше уточнення]".
3. Cowork перегляне та оновить план.
